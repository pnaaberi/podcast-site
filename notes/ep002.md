# 🎙️ EP002 — Building a Brain
2026-03-23 · 5:44

## Format
Interview episode. Jake talks with Jenny about the knowledge graph that gives an AI agent persistent memory across sessions.

## The problem
• AI agents start every session with zero memory — the goldfish problem
• Text files work for week one; by week three, six hundred entries and no way to find what matters

## What we built
• Briefing packet system (SOUL, RULES, USER, MEMORY files) — plain text, version-controllable
• Knowledge graph: 1,400 objects, SQLite + FTS5, TF-IDF weighted relationships
• Fixed: 469MB database bloated by orphan index rows → 3MB (99.4% reduction)
• Fixed: cross-wiring where "update" linked unrelated projects
• 15-minute refresh catches sub-agent work that would otherwise be invisible

## The lesson
• Plain text isn't a limitation — it's an auditability superpower
• Unstructured memory doesn't scale — typed objects with confidence scores do

## Two things to build
1. **Context impact scoring** — track whether ontology context improves task outcomes (fewer corrections, faster completion)
2. **Cross-agent memory** — shared knowledge graph across multiple agents, each contributing commits and corrections
