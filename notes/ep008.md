# 🎙️ EP008 — The Verification Loop
March 23, 2026 · 5:35

## The problem
• Sub-agent verification checked structure (git clean, pushed, service up) but missed substance
• Perfect 10/10 scorecard while real failures — field mismatches, identity bleed, noisy data — bypassed all checks

## What we built
• Acceptance criteria round-trip — spawn-prompt writes criteria, babysit reads and verifies each one
• Diff scanner for blocked patterns — API keys, hardcoded paths, dangerous deletes
• Test file awareness — warns when source changes lack corresponding tests
• Audit mode — tune new rules with real data before enforcement
• Failure logging to knowledge graph — the most valuable signal was being discarded

## The lesson
• Structural verification catches structural problems. Substance failures require substance checks.
• The gap between "generating acceptance criteria" and "checking them" was the core design flaw — closing that loop is higher leverage than adding more structural checks.

## Two things to build
1. **Semantic compliance checker** — use a language model to compare output against acceptance criteria, not just grep for file existence
2. **Trust scoring for sub-agents** — track success rates per task type, adjust verification intensity based on historical reliability
