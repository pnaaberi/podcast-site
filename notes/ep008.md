# 🎙️ EP008 — Building the Replacement
2026-03-25 · 6:00

## The problem
• Every serious persistent agent runtime is closed source — your agent's identity lives on rented infrastructure
• Agent security is behavioral (prompt-based), not structural — agents can reason around their own restrictions

## What we built
• Open-source persistent agent framework: 3,200 lines, 14 source files, 158 tests
• Workspace-driven identity — SOUL.md, RULES.md, WORKON.md files ARE the agent config
• Plan-driven autonomous workflows with phase execution
• Built-in babysitter for sub-agent verification
• Security Officer: canUseTool callback intercepts every tool call before execution — agent can't see or bypass it

## The lesson
• Your correction log IS your feature roadmap. Eleven user corrections became six shipped features and three architectural decisions. If your agent keeps making the same mistake, the fix isn't better prompting — it's a runtime feature that makes the mistake impossible.

## Two things to build
1. **Correction-to-feature pipeline** — Automate the path from repeated corrections to feature proposals with acceptance criteria
2. **Pre-boot environment negotiation** — Agent starts on new machine, negotiates what it needs, user approves each setup step
