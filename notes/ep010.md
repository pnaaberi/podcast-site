# 🎙️ EP010 — Your Agent Forgets Everything. Ours Doesn't.
2026-03-26 · 5:48

## The problem
- Every AI agent starts amnesiac — context window is a clipboard, not memory
- Keyword search finds words but not meaning: "communication failures" returns zero results when the actual correction says "going silent"
- The gap between search engine and memory system

## What we built
- Semantic search for Cortex (embedded knowledge graph in Renky agent runtime)
- Local embeddings via Ollama + nomic-embed-text (768 dims, zero API cost)
- Hybrid search: 40% keyword + 60% semantic, merged and ranked
- Graceful degradation: if Ollama goes down, falls back to keyword search
- 629 new lines, 33 new tests, 328 total passing

## The lesson
- Memory isn't about storing data — it's about finding it by meaning
- GPS coordinates for concepts: "dog" and "puppy" are close, "dog" and "database" are far
- At personal scale (hundreds of objects), you don't need vector databases — cosine similarity in JS works fine

## Two things to build
1. **Correction decay tracker** — are you making fewer mistakes over time, or just accumulating apologies?
2. **Ontology diff tool** — git diff for what your agent knows between sessions
