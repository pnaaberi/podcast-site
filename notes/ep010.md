# 🎙️ EP010 — The Knowledge Graph Inside the Machine
March 25, 2026 · 9:28

## The problem
• Every agent framework treats memory as an afterthought — flat files or vector DBs bolted on
• A standalone knowledge graph works, but HTTP calls to your own memory is like keeping your brain in a jar across the room

## What we built
• **Renky** — open-source personal agent runtime (3,000 lines, 295 tests, TypeScript)
• **Cortex Phase A** — embedded knowledge graph: 6 SQLite tables, FTS5, 11 object types, 10 link types, 70 tests
• **Cortex Phase B** — 3 auto-populating ingestors (git, workspace, services), 47 more tests
• **Security Officer design** — canUseTool runtime gate that the agent can't see or bypass

## The lesson
• Memory should live inside the agent process, not across a network boundary
• Objects with typed relationships beat both vectors (no structure) and plain tables (no search)
• Security locks belong on the outside of the door, not the inside

## Two things to build
1. **Correction decay tracker** — chart corrections per week by pattern to see if the agent is actually learning
2. **Ontology diff tool** — git-diff for knowledge graphs, showing what changed between sessions
