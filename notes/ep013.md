# 🎙️ EP013 — Poisoning Your Own Memory
2026-04-02 · solo

## The RAG poisoning chain
- Fetch a webpage → content enters knowledge graph → future query returns poisoned context
- The agent trusts its own memory — poisoned content looks like learned knowledge
- Persistent across sessions — one injection, permanent influence

## What we built
- Content sanitizer with 17 regex patterns across 5 categories
- Instruction overrides, persona hijacking, memory injection, exfiltration, encoded payloads
- Runs inside SecureCortex.ingest() — every write path is covered
- Dirty content gets [REDACTED], audit entry logged

## The deeper lesson
- Access controls decide who can write. Content controls check what gets written.
- Both are needed. Most frameworks only have access controls.
- 13 tests verify detection, cleaning, and property-level scanning

## Two things to build
1. **Anomaly detection** — flag content that's statistically different from existing Cortex objects
2. **Quarantine mode** — hold suspicious content for review before indexing
