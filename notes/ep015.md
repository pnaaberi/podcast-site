## OpenClaw vs Renky — same foundation
- Both: gateway + channels (Telegram/Discord) + memory (markdown) + crons + sub-agents + shell
- OpenClaw optimized for setup speed (5 min, went viral)
- Renky optimized for operational trust

## Where Renky diverges
- Memory: flat files vs Cortex knowledge graph (3,024 objects, semantic search)
- Security: prompt red lines vs deterministic code-level blocking (19 patterns)
- Verification: none vs babysitter (10 checks) + action verifier + skill tests
- Testing: none vs 1,097 tests, lint score 88/100
- Sessions: channel-locked vs export/import/handoff across devices

## The philosophy difference
- OpenClaw: accessible, fast setup, great marketing
- Renky: verifiable, enforceable, testable
- One scales to demos. The other scales to production.
