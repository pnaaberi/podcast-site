Three ideas. Workspace-driven identity — personality, rules, current task are markdown files in a directory. Edit a file, agent changes. No redeploy.

Plan-driven workflows — agent reads plan files, executes phases autonomously, hands off state between sessions through files.

Built-in verification — sub-agents get five automatic checks on completion. Git state, build, error scanning. Not optional.

Real-time accountability. Good. The Security Officer.

[JAKE]
Every framework puts security in the system prompt. "Ask before doing dangerous things." Behavioral security. The agent decides whether to follow it. Context gets long, those instructions get dropped. Lock on the inside of the door.

Our approach — a runtime callback before every tool execution. Agent doesn't see it. Can't reason about it. Tool calls classified against an allowlist. Approved or denied before the agent knows it tried. Four trust contexts with different permission ceilings.

[CHRIS]
Concept is sound. Implementation concerns me. Pattern matching on bash commands. Walk me through an attack.

[JAKE]
Real one from our stress tests. Allowlist blocks SSH commands. Pattern catches "ssh user at host." But "bash -c" with a base64-encoded SSH command inside? Pattern matcher sees "bash -c" and a string. Doesn't know what's inside.

[CHRIS]

One line the agent would have to deliberately construct to avoid detection. Meaning the agent is actively adversarial. Tier one catches accidents and obvious violations. Tier three escalates to the human with auth codes. Defense in depth, not a single wall.

That model upgrades independently. But you're right this is a race, and pattern matching won't win long-term.
