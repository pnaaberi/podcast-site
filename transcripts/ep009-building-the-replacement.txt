[JENNY]
Tell me about the Security Officer. This sounds novel.

[JAKE]
Every agent framework handles permissions the same way. Instructions in the system prompt saying "ask before doing dangerous things." That's a lock on the inside of the door. The agent sees it. Can reason about it. Can decide to ignore it. Context overflow can silently evict those safety rules.

We put the lock on the outside. A callback fires before every tool execution. The agent never sees it. Can't manipulate it. Can't reason around it. Every tool call gets classified, checked against an allowlist, approved or denied before the agent knows it tried.

[JENNY]
Different trust levels?

[JAKE]
Four contexts. Interactive — user actively chatting, most permissions. Autonomous — agent on a cron job, tighter. Sub-agent — spawned worker, even tighter. Sub-agent autonomous — the most locked down. Can't touch system access, can't modify infrastructure, can't delete outside its workspace. The agent doesn't get a vote.
