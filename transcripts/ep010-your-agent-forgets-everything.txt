That's the gap we closed today. Semantic search. Instead of matching words, you match meaning.

You take text and turn it into a list of numbers — an embedding. GPS coordinates for meaning. "Dog" and "puppy" are close together. "Dog" and "database" are far apart. Search becomes: turn the query into coordinates, find the closest match.

[ALEX]
The embedding model?

[JAKE]
Local. Open-source model, running on our hardware through Ollama. No API calls, no costs, no data leaving the machine. Seven hundred sixty-eight dimensions. About a second per embedding.

[ALEX]
Storing vectors in SQLite without a vector extension?

[JAKE]
Binary blobs. Load into memory, cosine similarity in JavaScript. At three hundred objects, instant. At ten thousand, we'd need an index. We're at three hundred.

[SAM]
What happens when the embedding service goes down?

[JAKE]
Nothing breaks. The function returns null. Semantic search returns empty. The system falls back to keyword search. The agent gets slightly worse results until it comes back. No crash, no error, no difference from the agent's perspective.

[SAM]
How do the two search methods combine?

[JAKE]
Hybrid search. Every query runs both. Results merge, deduplicate, rank. Forty percent keyword weight, sixty percent semantic. If one method catches something the other missed, it surfaces.

Objects get embedded automatically on ingest. Fire-and-forget — the object saves immediately, embedding happens in the background.
